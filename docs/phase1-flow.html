Règles de transition — State Machine

Aucun incident consecutive < retry_count → no-op (attente)
Aucun incident consecutive ≥ retry_count → OPEN (severity = medium/high/critical)
OPEN / ACK nouvelle failure → no-op (update severity si escalade)
OPEN / ACK résultat OK → RESOLVED + resolved_at = now()
RESOLVED nouvelle failure streak ≥ threshold → nouvel OPEN (nouveau UUID)

Severity mapping (× retry_count threshold)

1× threshold→ medium
2× threshold→ high
3× threshold→ critical
